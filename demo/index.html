HiGHS for JavaScript.

The open-source HiGHS optimization solver, packaged for Node.js and modern browsers. Build and solve linear, mixed-integer, and convex quadratic models from JavaScript.

npm install highs Node.js 18+

// Solve a model in CPLEX LP format

LP · MIP · convex QP

What it solves

Three useful classes of optimization problem.

HiGHS is designed for models whose objective and constraints are linear, plus continuous convex quadratic objectives. Choose the class that matches which variables must be discrete and whether the objective contains quadratic terms.

LP

Linear programs

Continuous decisions with a linear objective and linear row bounds. HiGHS provides revised simplex, interior-point with crossover, and PDLP algorithms.

MILP

Mixed-integer programs

Linear models with integer, binary, or semi-continuous columns. HiGHS uses branch-and-cut with presolve, heuristics, and gap-based stopping.

QP

Convex quadratic programs

Continuous columns, linear constraints, and a convex quadratic objective supplied as a sparse Hessian. Non-convex QP and MIQP are outside HiGHS’ scope.

LP // ONE-SHOT CPLEX TEXT SOLVER highs.solve()

Solve a model from LP text

Use this entry point when a model already exists in CPLEX LP format, or when generating readable model text is simpler than assembling sparse arrays. solve() parses and solves the text in one call, then returns a name-keyed result with the objective, column values, row values, duals, and basis statuses. It reparses the model on every call; use a persistent model for repeated solves or incremental changes.

QP // QUADRATIC OBJECTIVE & HESSIAN MATRIX passHessian()

Markowitz Portfolio Risk Minimization

Use a convex quadratic program when cost or risk depends on interactions between continuous decisions, such as portfolio covariance or least-squares penalties. This model minimizes portfolio variance while meeting a target return. HiGHS evaluates ½xᵀQx + cᵀx, so Q is twice the covariance matrix here. The Hessian must have the correct semidefinite sign for the objective sense; HiGHS does not solve non-convex QP or mixed-integer QP.

RANGING // POST-OPTIMAL SENSITIVITY ANALYSIS getRanging()

Objective Cost & Constraint Sensitivity

Use ranging after an LP solve to ask how far one objective coefficient or row bound can move while the current optimal simplex basis remains optimal. HiGHS reports independent up/down ranges for column costs and row or column bounds; these are local, one-at-a-time basis-stability intervals, not guarantees for simultaneous changes. Ranging requires an optimal LP basis and is not available for MIP or QP solutions.

OPTIONS // SOLVER TUNING & CALLBACK HOOKS options.describe()

HiGHS Parameter Introspection

Use options to control stopping criteria, feasibility tolerances, presolve, algorithm selection, logging, and MIP search behavior for a persistent model. options.describe() returns the native type, current value, default, and numeric range so applications do not need to hard-code metadata. The WebAssembly API rejects thread/concurrency settings and options that write to native filesystem paths.

IIS // INFEASIBILITY CONFLICT DIAGNOSTICS getIis()

Explain why a model is infeasible

Use an irreducible infeasible subsystem (IIS) when a model has no feasible solution and you need a smaller set of rows and variable bounds to inspect. “Irreducible” means every returned member is necessary for that particular conflict: removing any one makes the subsystem feasible. It does not mean HiGHS found the conflict with the fewest possible members, and a model can contain several different IISs.

I/O // IN-MEMORY SERIALIZATION & NORMALIZATION exportModel("lp")

Import, inspect, and export models

Use in-memory model I/O to accept LP or MPS data from uploads, generated files, or another service, then solve or convert it without exposing a native filesystem path. LP is readable text; MPS preserves a more rigid interchange structure and is represented as bytes where required. Exported models are normalized representations of HiGHS’ internal model, so comments and original formatting are not preserved.
